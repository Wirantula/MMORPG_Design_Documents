# Contributing
Use contract-first, test-backed changes aligned with the backlog and ADR process.

## Branch naming
- `feat/<story-id>-<short-name>`
- `fix/<story-id>-<short-name>`
- `chore/<short-name>`

## Commit style
Use conventional commit prefixes:
- `feat:`
- `fix:`
- `refactor:`
- `test:`
- `docs:`
- `chore:`

## Pull request requirements
- Story reference (`Epic.Story` or ticket ID)
- Acceptance criteria evidence
- Test evidence
- Observability impact note
- Rollback/operational note for risky changes
