# Story 4.1: World Time Service

## Problem / intent
The simulation needs a stable, configurable world clock that maps real time to accelerated in-game time (1:30 ratio), exposes snapshots to clients via periodic ticks, and tracks drift/lag for operational monitoring.

## Acceptance criteria
- World time advances at a configurable acceleration factor (default 30x).
- World seed time is configurable via environment variable.
- A tick loop runs at a configurable interval and broadcasts world snapshots to all connected WS clients.
- Game-day transitions emit a `TickCompleted` domain event.
- Simulation metrics (tick count, tick duration, current game day, drift) are exposed via the Prometheus `/api/metrics` endpoint.
- All existing tests continue to pass; new tests cover configurable clock, world time math, game-day calculation, tick loop behaviour, and metric recording.

## Dependencies
- Story 1.3 (observability baseline) — completed.

## Implementation notes
- `SimulationService` enhanced with `getWorldTimeMs()`, `getGameDayNumber()`, configurable constructor.
- `TickService` manages `setInterval`-based loop, day-change detection, WS broadcast, and metric push.
- `ObservabilityService` extended with `recordSimulationMetrics()` and five new Prometheus gauges/counters.
- New env vars: `WORLD_SEED_ISO`, `TIME_ACCELERATION`, `TICK_INTERVAL_MS`.
- `DomainEventBus` added as shared in-process event infrastructure.

## Test notes
- `test/simulation.service.test.ts` — 7 tests covering defaults, configurable acceleration/seed, world-time math, game-day indexing.
- `test/tick.service.test.ts` — 7 tests covering tick counting, day-change events, metric propagation, action resolution during tick.
- `test/domain-events.test.ts` — 4 tests for event bus subscribe/emit/remove/isolation.

## Observability notes
- New Prometheus metrics: `cybaworld_simulation_tick_count`, `cybaworld_simulation_tick_duration_ms`, `cybaworld_simulation_tick_max_duration_ms`, `cybaworld_simulation_current_game_day`, `cybaworld_simulation_drift_ms`.
- Structured log on each game-day transition and tick loop start.
