How this plan should be used
This document converts the design into an executable product plan. It is structured so that a human product owner can steer the work while an agentic development team can execute in parallel with controlled interfaces. The plan assumes iterative delivery with vertical slices, hard definitions of done and explicit dependency mapping.

3. Working rules for Codex-oriented delivery
• Every story must have acceptance criteria, technical notes, dependency tags and a clear owner.
• Agents may work in parallel only when a contract boundary is already defined.
• Schema and message changes require contract-first review before implementation spreads.
• Each meaningful change must update tests, docs and at least one observability hook.
• Never allow AI-generated code to publish direct game balance changes without explicit review.
• Keep AGENTS.md, ADRs and story templates in-repo so context survives across sessions.

6. Definition of done
• Feature behavior matches accepted story scope.
• Required tests are present and passing.
• Docs, contracts and ADR references are updated.
• Logs, metrics and moderation implications are considered.
• No unchecked TODOs remain in merged code.
• A human reviewer has verified that AI-generated output is appropriate.
7. Recommended repo documents
• docs/GDD.md - design ground truth
• docs/BACKLOG.md - current epics and story status
• docs/AGENTS.md - instructions, repo map, agent roles and handoff conventions
• docs/ADR/ - architecture decisions
• docs/RUNBOOKS/ - operational and incident procedures
• docs/BALANCE/ - economic and progression tuning notes

9. Final delivery guidance
Do not attempt to build the entire MMO horizontally before a vertical slice exists. The first meaningful milestone is not 'database complete' or 'all systems stubbed'; it is a tiny but runnable world where one account can create one life, spend time, produce state changes and survive through a short simulated routine. Every later epic should protect that vertical path instead of burying it.
